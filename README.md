# HolyFake Staff Control Panel — финальная версия

Локальное веб-приложение для учета сотрудников КП HolyFake: сотрудники, наказания, отпуска, повышения, аттестации, собеседования, ЧСКП/ЧСП, архив, уведомления, журнал действий, импорт/экспорт Excel и резервные копии.

## Быстрый запуск на Windows

1. Распакуйте архив.
2. Откройте папку `holyfake_cp_app`.
3. Запустите `run.bat`.
4. Откройте: http://127.0.0.1:8000

## Ручной запуск

```bash
cd holyfake_cp_app
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
uvicorn main:app --reload
```

## Первый вход

В финальной версии оставлен только один системный аккаунт владельца:

| Роль | Логин | Email | Пароль |
|---|---|---|---|
| owner / главный администратор | `owner` | `owner@holyfake.local` | `HolyFake#2026!` |

После первого входа лучше сразу поменять пароль owner в **Настройки → Пользователи админ-панели**.

## Как выдавать доступ сотрудникам

1. Войдите под `owner`.
2. Откройте **Настройки → Пользователи админ-панели**.
3. Создайте пользователя:
   - логин: ник в Minecraft, например `Zomik_`;
   - email: можно реальный email или технический, например `zomik@holyfake.local`;
   - пароль: задаете сами и передаете сотруднику;
   - роль: выбираете нужную.
4. Сотрудник заходит на сайт по своему логину или email и паролю.

## Как забрать доступ у уволенного сотрудника

Самый быстрый способ:

1. **Настройки → Пользователи админ-панели**.
2. Найти пользователя.
3. Нажать **Заблокировать**.

Также добавлена автоматизация: если сотрудника снимают через кнопку **Снять** и логин пользователя сайта совпадает с ником сотрудника, доступ к сайту блокируется автоматически. Поэтому лучше создавать логин сайта точно таким же, как ник сотрудника в Minecraft.

Управлять пользователями, менять роли, менять чужие пароли и блокировать доступ может только `owner`.

## Роли

- `owner` — полный доступ, создание пользователей, блокировка, смена ролей, настройки, импорт/экспорт, резервные копии.
- `admin` — управление данными панели, настройки, импорт/экспорт, резервные копии, но без управления пользователями.
- `moderator` — рабочие действия: сотрудники, наказания, отпуска, повышения, аттестации, собеседования, ЧСКП/ЧСП.
- `staff` — просмотр без важных изменений.

## Черные списки

Есть два раздела:

- **ЧСКП** — черный список команды проекта;
- **ЧСП** — черный список проекта.

При добавлении или редактировании сотрудника система проверяет совпадения по:

- ник;
- Discord;
- Discord ID;
- Telegram;
- Telegram ID;
- почта.

Если есть совпадение хотя бы по одному полю, показывается красное предупреждение и запись сохраняется в журнал действий.

## Аттестация

Иерархия аттестации:

```text
Стажер → Мл.сотрудник → Сотрудник → Вед.сотрудник → Ст.сотрудник / Спектатор
```

Правила:

- нельзя сдавать на должность ниже или равную текущей;
- после неудачной попытки ставится КД 2 дня;
- после 3 неудачных попыток подряд ставится задержка 7 дней;
- при успешной аттестации сотрудник автоматически повышается до выбранной должности;
- успешная аттестация сохраняется в истории аттестаций.

## Резервные копии

В разделе **Резервные копии** можно:

1. создать универсальный файл `.json`;
2. скачать его;
3. загрузить ранее скачанный файл и восстановить базу.

JSON-копии работают и локально с SQLite, и на Render с Neon/PostgreSQL. Старые `.sqlite3`-копии тоже можно загрузить: сайт попробует перенести данные из них в текущую базу. Перед восстановлением текущая база автоматически сохраняется как аварийная копия.

## Импорт таблиц

Поддерживается импорт `.xlsx`, `.xlsm`, `.csv`, а также Google Sheets по ссылке.

Если Google Sheets возвращает ошибку `401 Unauthorized`, таблица закрыта. Надежнее скачать таблицу через **Файл → Скачать → Microsoft Excel (.xlsx)** и импортировать файл.

## Хостинг

Для локального использования ничего выкладывать не нужно: сайт доступен только на вашем ПК по адресу `127.0.0.1:8000`.

Для использования 4–6 людьми через интернет можно разместить проект на Render, а базу держать в Neon PostgreSQL.

На Render обязательно добавьте переменные:

- `DATABASE_URL` — строка подключения Neon PostgreSQL;
- `HF_SECRET_KEY` — длинный секретный ключ для сессий.

Локально без `DATABASE_URL` сайт автоматически использует SQLite-файл `holyfake.sqlite3`. На Render с `DATABASE_URL` данные сохраняются во внешнюю PostgreSQL-базу и не пропадают после redeploy.

Стартовая команда для хостинга:

```bash
uvicorn main:app --host 0.0.0.0 --port $PORT
```

## Безопасность

- пароли хранятся только в хешированном виде;
- закрытые страницы защищены на backend;
- при блокировке аккаунта пользователь теряет доступ;
- есть ограничение попыток входа;
- сессия хранится в защищенной cookie;
- для хостинга обязательно задайте свой `HF_SECRET_KEY`.

## Права доступа по пользователям

В этой версии можно точечно настроить доступ каждому пользователю сайта:

1. Зайдите под `owner`.
2. Откройте **Настройки → Пользователи админ-панели**.
3. Создайте пользователя с логином = ник в Minecraft.
4. Ниже откройте карточку этого пользователя в блоке прав.
5. Отметьте галочками:
   - какие разделы он видит;
   - какие действия он может выполнять.
6. Нажмите **Сохранить права**.

Если нажать **Сбросить до роли**, ручные права удалятся и снова применятся стандартные права выбранной роли.

`owner` всегда имеет все права и не может быть ограничен. Это сделано специально, чтобы не потерять управление сайтом.

## Чистая база

Финальная сборка идет без тестовых сотрудников, без тестовых ЧСКП/ЧСП, без тестовых собеседований и без фейковых ников в списках. Оставлены только:

- системный аккаунт `owner`;
- должности;
- статусы сотрудников;
- `Система` в списке выдающих наказания.

Списки «Кто принимает», «Кто проводит собеседования» и другие значения можно заполнить вручную в **Настройки**.
